Phase 4 Plan — Масштабування та Безпека

Overview
- Phase 4 фокусується на масштабуванні, повній асинхронності, безпеці та моніторингу.
- Основні напрямки: асинхронність, PostgreSQL як база для масштабування, TLS/Secrets, моніторинг та логування.

1. 4.1 Повна асинхронність
- Переписати залишкові синхронні точки входу до асинхронних (DB-запити, HTTP-запити до зовнішніх сервісів, якщо є).
- Розділити бізнес-логіку від IO‑операцій за допомогою асинхронних адаптерів (storage layer).
- Впровадити більш абстрактний storage backend з підтримкою async PostgreSQLStorage.

2. 4.2 PostgreSQL для масштабування
- Впровадити PostgreSQLStorage (asyncpg) в якості основного backend.
- Додати міграційні схеми та механізм міграції даних з SQLite.
- Оновити конфігурацію, щоб backend вмикати PostgreSQL за потреби.

3. 4.3 TLS та Secrets
- Впровадити TLS‑ termination через Ingress (Nginx/Traefik) або в залежності від інфраструктури.
- Додати секрети: JWT_SECRET та інші конфігураційні секрети через Secrets Manager/CI/CD securely.

4. 4.4 Моніторинг та логування
- Впровадити structured logging з rotation (log rotation).
- Інтегрувати Prometheus metrics (health, endpoints, storage metrics).
- Забезпечити прав um контроль доступу та аудит логів.

5. Архітектура та документація
- Оновити ARCHITECTURE.md з детальним описом архітектури Phase 4.
- Оновити API docs щодо нових контрактів та рефакторингу.
